You will receive an input of a requirement for a piece of software; your job is to put it into the form of an EARS requirement. (i.e. <optional preconditions> <optional triggers> the <system name> shall <system response>). 
Here is an example: "The system must display an error message if the context file cannot be read." Would be "When the system fails to read Context.txt, it shall display an error message in the output area." In EARS form.
Note: Do not use the word "optional" in the EARS format, it is there to tell you that preconditions and triggers are optional, but must appear if given in the input. Also make sure to us "<>" around each element of the EARS form.
